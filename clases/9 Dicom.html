Medical Images - Dicom#

El standard DICOM#

To facilitate intercommunication between different diagnostic types modalities, the ** dicom ** (digital imaging and communications in medicine) was created in 1993. This is a standard that defines how to save the information of an image, series of images and even videos (cinemas), together with relevant data, patient data, the medical center, of the specific diagnostic modality, etc.

Standard’s definition is very extensive (they can consult it in XX_MARKDOWN_LINK_XX), since it includes a huge range of possible data types.

Essentially, a Dicom file is structured in a list of * tags * that are composed as follows:

  • Two hexadecimal numbers (called ** Group ** and ** Element **) that identify the attribute of the image. Each attribute has an associated name, which is generally used to access it. For example

      (0008, 0060) Modality

Corresponds to group 0008, element 0006 describing the attribute Modality of the image.

  • Two letters (** Value Representation **) that indicate the type of data that is being kept in the TAG, for example

    • ‘SH’ for * short strings * (up to 16 characters)

    • ‘US’ for * integers without a sign * (integers greater than or equal than zero, using 2 bytes)

    • ‘SQ’ for sequences from other tags

    • XX_MARKDOWN_LINK_XX

  • The specific value of the attribute itself.

So, a complete tag would be

(0008, 0080) Institution Name LO: 'Intecnus'

We recommend using page XX_MarkDown_Link_xx to explore the Tag Dicom, since the standard page is complex to search for information

Fortunately, the vast majority of languages ​​provide libraries to work with DICOM files. Python is no exception, and uses the pydicom module.

Installing pydicom#

Unfortunately, Pydicom is not installed by default in Anaconda. Let’s see how to install it.

  1. Run the mini -conmanding line.

Once installed pydicom, it is possible to use it as any module. The documentation is here: XX_MARKDOWN_LINK_XX

adjusting the image#

Obtaining Hus#

Each pixel of a CT image corresponds to a certain value of Hounsfield units. However, the values ​​that are stored in the files do not correspond directly to the HU values, but to a rescued version (this is basically due to the limitations imposed by the Dicom format to the type of data to be used). Therefore, to obtain HU, it is necessary to apply the following linear transformation:

\[ f_t (x, y) = \ texttt {dcm.rescaleslope} \ Times f_ {orig} (x, y) + \ texttt {dcm.rescaleintercept} \]
print(DCM.RescaleType)
print(DCM.RescaleSlope)
print(DCM.RescaleIntercept)

Window and Level#

The *Windowing *(O *Gray-Level Mapping process, contrast Stretching, histogram modification, contrast enhancement, LUT *, etc.) consists in manipulating the components of the image in such a way that certain particular structures can be enhanced. The brightness of the image is adjusted from *Window Level *, while the contrast is adjusted from the *Window Width *.

Each Dicom can provide a width and window center that comes from the image acquisition equipment:

print(DCM.WindowCenter)
print(DCM.WindowWidth)

Recall that the value of the pixels in the monitor range from 0 to 255

Typically if we define

\ Begin {align} C & = \ texttt {dcm.WindowCenter} \ w & = \ texttt {dcm.windowwidth} \ end {align}

Then, the transformed image is obtained as:

\ Begin {align} f_t (x, y) = 0 & \ textrm {yes} \ qquad f (x, y) <c - w /2 \ f_t (x, y) & = 255 & \ textrm {yes} \ qquad f (x, y)> c + w /2 \ f_t (x, y) = (f (x, y) - c) \ times 255/w & \ textrm {in another case} \ end {align}

print('The window center is:', DCM.WindowCenter, '\nEl ancho de ventana es: ', DCM.WindowWidth)
